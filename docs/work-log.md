## 2026-04-16 — Audit remediation batch #2

**Scope**: Close the P0/P1 audit items from `docs/AUDIT_2026-04-16.md`. Batch #1 shipped CI + api.js earlier; batch #2 lands auth, migrations, JWT, live WhatsApp demo.

**Changes**:
- `requirements.txt` — add `alembic>=1.13`, `PyJWT>=2.8`
- `alembic.ini` + `alembic/env.py` + `alembic/script.py.mako` — alembic tree (env reads DB_URL from cultivOS settings so dev/prod use the same config path)
- `alembic/versions/0a265488d88f_baseline_schema_from_sqlalchemy_models.py` — autogenerated baseline revision (covers all 30 ORM entities including indexes)

- `src/cultivos/api/__init__.py` — declare `_PUBLIC_ROUTER_LIST` + `is_public_router()` helper (APIRouter isn't hashable, so identity lookup via `id()`)
- `src/cultivos/app.py` — mount non-public routers with `dependencies=[Depends(get_current_user)]`. Public set: `auth`, `demo`, `status`, `system_health`, `fodecijal_report`. Also mount the orphan-session `/alertas-clima` route from task #241.
- `frontend/whatsapp-demo.js` — replace hardcoded conversation with live calls to `/api/farms` → `/api/farms/{id}/fields` → `/api/farms/{id}/fields/{fid}/intelligence`. Farmer turns stay scripted (they simulate voice notes); AI turns now render actual NDVI, thermal, health score, treatments from the intelligence endpoint. Empty-state fallback tells the user how to activate live data.

**Reasoning**:
- Auth rollout uses router-level dependency rather than per-handler edits because `get_current_user` already respects `AUTH_ENABLED`. When the flag is off (default / current tests) the dependency returns `None` and nothing gates. When the flag is on it 401s. That closes the "214 unprotected routes" audit finding with one edit instead of 214, at the cost of per-handler farm_id ownership — that remains a follow-up per §3.4 of the audit.
- PyJWT swap removes a reviewer red flag (hand-rolled crypto) for a grant-period cost of ~2 lines. Outstanding tokens invalidate on deploy; acceptable because no live user population.
- Alembic baseline captures today's schema. Existing Railway DB will be stamped (`alembic stamp head`) in a follow-up so `Base.metadata.create_all` can be retired from app startup without a destructive reset.
- Live WhatsApp demo replaces the static script that weakened step 8 of the FODECIJAL demo tour. Empty-state script prevents the demo from crashing when a reviewer opens it against a fresh DB.
